# 环球模式穷逼方式修复说明

## 问题描述
在使用环球模式（mode=0）的穷逼方式（rich_mode=1）时，自动进入招募页面后的连续点击过程无法持续，导致隔一段时间就退出并重新进入招募，形成"进-出-进-出"的循环，影响效率。

## 根本原因
原始代码中 `find_click_recruitment()` 函数使用固定的 `for i in range(20):` 循环，只能执行20次连击后就自动退出，导致频繁的循环中断和重新进入。

## 解决方案

### 1. 修改 `find_click_recruitment()` 函数
**改动点：**
- ✅ 将有限的20次循环改为动态的无限循环（`while` 循环）
- ✅ 添加进入成功检测：当检测到 `find_in_huanqiu_team()` 返回True时，立即退出循环
- ✅ 添加最大尝试次数限制（200次）作为安全门槛，防止极端情况下的无限卡顿
- ✅ 增强日志输出，显示当前循环次数和每次点击的按钮数量

**核心逻辑流程：**
```
1. 首先进入招募页面
2. 开始无限连击循环：
   ├─ 检查是否已进入环球队伍 → 是则退出循环
   ├─ 截图并查找所有环球按钮
   ├─ 快速点击所有找到的按钮
   ├─ 检查并处理离开按钮
   └─ 重复直到成功
3. 成功进入环球队伍后函数返回
```

### 2. 优化主循环中的穷逼模式逻辑
**改动点：**
- ✅ 简化流程，因为 `find_click_recruitment()` 现在已确保成功进入
- ✅ 添加3秒等待时间，让环球队伍完全加载
- ✅ 改进后续错误处理

**新流程：**
```
穷逼模式执行流程：
1. 调用 find_click_recruitment()
   └─ 自动进入招募并无限连击直到成功进入环球队伍
2. 等待队伍组建完成（3秒）
3. 尝试点击开始游戏按钮
4. 如果失败，返回重新循环
5. 成功则进行后续游戏
```

## 关键改进点

| 方面 | 原始行为 | 改进后 |
|-----|--------|------|
| **连击持续性** | 固定20次后停止 | 无限连击直到成功 |
| **进入检测** | 被动等待 | 主动检测 `find_in_huanqiu_team()` |
| **安全性** | 无上限 | 最多200次尝试 |
| **日志可读性** | 较少信息 | 显示循环次数和点击数 |
| **流程效率** | 频繁中断和重新进入 | 连贯的单一流程 |

- 如果超过50次仍未成功，可能存在模板匹配问题

## 可能的调整参数

如果需要微调性能，可以修改以下参数：

### 在 `find_click_recruitment()` 函数中：
- `max_attempts = 200`：修改此值调整最大尝试次数
- `time.sleep(0.05)`：修改此值调整连击间隔（更小=更快）

### 在主循环中：
- `time.sleep(3)`：队伍加载等待时间，可根据需要调整（1-5秒）

## 回滚方案

如需恢复原始行为，修改部分如下：

1. 将 `while attempt_count < max_attempts:` 改为 `for i in range(20):`
2. 删除进入成功检测逻辑
3. 恢复原始的主循环逻辑

## 已知限制

- 需要准确的模板图像匹配，确保 `huanqiu-*.png` 模板文件正确
- 网络延迟可能影响实际进入时间
- 游戏界面变化可能需要调整模板或参数
